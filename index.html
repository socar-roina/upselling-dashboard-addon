1. 일별 결제 건수 / 순매출 (예약 단위, 첫 결제일 기준)
예약과 상품 단위로 전 기간 amount를 합산해 순액 양수만 결제로 집계
날짜는 첫 결제일 기준이라 일자 경계 중복 없음
WITH per_rsv_type AS ( SELECT reservation_id, charge_type, SUM(amount) AS net_amount, DATE(MIN(IF(amount>0, created_at, NULL)),"Asia/Seoul") AS first_charge_dt FROM `socar-data.tianjin_replica.charged_info` WHERE DATE(created_at,"Asia/Seoul") >= "2026-06-05" AND charge_type IN ("addon_carseat","addon_stroller") GROUP BY reservation_id, charge_type HAVING net_amount > 0 ) SELECT first_charge_dt AS dt, COUNTIF(charge_type="addon_carseat") AS carseat_cnt, SUM(IF(charge_type="addon_carseat", net_amount,0)) AS carseat_rev, COUNTIF(charge_type="addon_stroller") AS stroller_cnt, SUM(IF(charge_type="addon_stroller", net_amount,0)) AS stroller_rev, COUNT(DISTINCT reservation_id) AS rsv_cnt, SUM(net_amount) AS total_rev FROM per_rsv_type GROUP BY dt ORDER BY dt
2. 결제 후 취소, 환불 예약 검증 (net=0의 정체)
순액 0 예약을 state, cancel_at, 결제~환불 시간차로 분해
전부 cancel_at 존재로 취소된 예약
WITH rsv AS ( SELECT reservation_id, SUM(amount) AS net_amount FROM `socar-data.tianjin_replica.charged_info` WHERE DATE(created_at,"Asia/Seoul") >= "2026-06-05" AND charge_type IN ("addon_carseat","addon_stroller") GROUP BY reservation_id ), zero_rsv AS ( SELECT reservation_id FROM rsv WHERE net_amount = 0 ), gap AS ( SELECT c.reservation_id, TIMESTAMP_DIFF(MAX(IF(c.amount<0,c.created_at,NULL)), MIN(IF(c.amount>0,c.created_at,NULL)), MINUTE) AS diff_min FROM `socar-data.tianjin_replica.charged_info` c JOIN zero_rsv z USING(reservation_id) WHERE c.charge_type IN ("addon_carseat","addon_stroller") GROUP BY c.reservation_id ) SELECT ri.state, COUNT(*) AS rsv_cnt, COUNTIF(ri.cancel_at IS NOT NULL) AS has_cancel_at, ROUND(AVG(g.diff_min),1) AS avg_refund_min FROM gap g JOIN `socar-data.dbt.stg_tianjin_replica__reservation_info` ri ON ri.id = g.reservation_id GROUP BY ri.state ORDER BY ri.state

용어 정리

결제 건수같은 예약의 부가서비스 청구와 환불을 합산해 순액이 양수인 예약 수
카시트와 유모차는 청구 타입(addon_carseat, addon_stroller)으로 구분
순매출결제액에서 환불액을 뺀 순 결제 금액
예약별 순액이 양수인 건만 합산
결제 후 취소 예약결제 후 같은 금액이 환불돼 순액이 0이 된 예약
전부 예약 자체가 취소돼 부가서비스가 함께 환불된 건
순매출과 결제 건수에서 제외
옵션 화면 노출결제 직전 부가서비스 옵션 화면(services_option)을 본 순 사용자 수
앱 옵션 추가 버튼결제 화면(checkout_reservation)에서 부가서비스 추가 버튼을 클릭한 행위
D+1 적재앱과 웹 이벤트 로그는 하루 뒤 배치 적재라 당일 데이터 확인 불가
결제(charged_info)는 실시간 적재
